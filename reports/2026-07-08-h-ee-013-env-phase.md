# 2026-07-08 - H-EE-013 Env-Derived Phase

## Plain-English Summary

We tested whether feeding the state-BC MLP its progress/phase features from **live
environment state** (distance bins, gripper opening, contact, lift) instead of the
open-loop demo `cursor` would fix EE event-order failures.

It did **not**. Under the same protocol-v2 **validation** grid and the same MLP training
contract used for H-EE-012 model selection, env-derived phase made both action spaces
**worse** than the selected `legacy_progress_phase` baseline:

| Contract | EE success | Joint success | Combined |
|----------|------------|---------------|----------|
| `legacy_progress_phase` (selected) | **31/120** | **53/120** | **84/240** |
| `none` (H-EE-012 cursor-free) | 32/120 | 18/120 | 50/240 |
| `env_derived_phase` (H-EE-013) | **19/120** | **23/120** | **42/240** |

**Verdict: H-EE-013 rejected** on registered validation. The final holdout was **not**
accessed. The selected shared temporal contract remains `legacy_progress_phase`.

Train loss looked fine (normalized MSE ~0.001–0.002). That is **not** evidence of rollout
success.

## Implementation Details

### Temporal contract

Added `TEMPORAL_FEATURE_MODES` entry `env_derived_phase`:

- Same **50-D** input layout as `legacy_progress_phase` (progress polys + phase one-hot +
  phase-progress polys).
- **Different source** of those scalars:
  - **Train:** `estimate_env_phase(observation_features)` on each demo sample (not demo
    `phase` / `step_index`).
  - **Rollout:** same estimator on live features; **cursor is ignored** for MLP features
    (cursor-invariant, like `none`).
- Global progress maps env phase index + phase progress through median
  `group_phase_lengths` onto the demo length scale (`env_phase_global_progress`).

### Estimator (Markov)

Priority order from live features only:

1. Contact + lift ≥ 0.028 m → `hold`
2. Contact + closed grip + lift rising → `lift`
3. Near object (≤ 0.025 m) and jaw not fully open, or closed contact before lift → `close_gripper`
4. Distance ≤ 0.022 m, open grip → `grasp_align`
5. Distance bins → `approach_2` / `approach_1` / `approach_0`

Distance-gates jaw-open state so far approach with a partially open home jaw is **not**
labeled `close_gripper`.

This is feature engineering / inductive bias, not new sensors. Phase is a deterministic
function of quantities already in the base observation.

### What was deliberately not done

- No inference-only patch of a legacy-trained model (H-EE-010 lesson).
- No distance-guard shield (H-EE-011 / diagnostic-only).
- No Phase 6b vision policy / VLA training.
- No final-split access.

## Decisions Made

- Decision: Train and evaluate under `env_derived_phase` with **identical** estimator.
  Reason: H-EE-010 showed inference-only feature changes are OOD / unfair.
- Decision: Use protocol-v2 **validation** only; do not open final.
  Reason: Same selection discipline as H-EE-012; final is one-shot for selected contracts.
- Decision: Match registered hyperparams (128×128, 300 epochs, 5 seeds).
  Reason: Otherwise confounded with capacity/epochs (first diagnostic run proved this).
- Decision: Reject H-EE-013; keep `legacy_progress_phase` as shared comparison contract.
  Reason: Combined validation success 42/240 ≪ legacy 84/240; EE event-order and early-close
  both degraded.
- Decision: Leave frozen `evidence/phase5_v2_model_selection.json` / final results untouched.
  Reason: H-EE-013 is a rejected candidate, not a new selected mode.

## Risks And Limitations

- Risk: Estimator thresholds are hand-tuned on scripted demos.
  Why it matters: A different binning might change outcomes; rejection is for *this* design.
- Risk: Env phase is a pure function of base observation features.
  Why it matters: Phase one-hots may be redundant transforms; the MLP may not gain new
  information and may get a sharp, wrong discrete mode when distance/contact flicker.
- Risk: No monotonic phase latch at rollout.
  Why it matters: Markov labels can chatter at phase boundaries (close ↔ grasp_align).
- Risk: Joint also regressed, so a shared env-phase contract cannot be selected for fair
  EE-vs-joint comparison even if EE had improved slightly.
  Why it matters: Action-space comparison still requires one shared temporal contract.
